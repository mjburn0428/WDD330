Week 08 Learning Notes

This week I learned about JSON webtokens or JWT for short. JWT are an open standard which defines a compact and self contained way to securely transmit information and data between parties which operates as a JSON object. The information can be verified and trusted because of a digital signature. JWT's are signed using a HMAC or a public or private key using RSA.

Programming basics: When to use JWT's

    When JSON webtokens are useful when one needs to have an Authentication method and exchange of information. Authentication is the typical scenario for when to use a JWT once a user is logged in. Each request will incldue the JWT allowing the user to access services, routes, resources which are permitted within the token. Single Sign On is a feature which uses JWT a lot. Information Exchange JWT's are an excellent way of securly transmitting information between users which is effective because it is signed.
