Week 01 Learning

This week I learned a bit about Git and what the functions are used when using Git. So Git is a tool that performs a function of code version control in a distributed way, of which several features stand out, such as:

  • Quick branch management, because Git predicts that a change would be merged far more frequently than it was originally planned.
  • Quick branch management, because Git predicts that a change would be merged far more frequently than it was originally planned.
  • Distributed management; Changes are imported as extra branches, which can be merged in the same way that the local branch can.
  • Project management that is effective.
  • Package replenishing on a regular basis.

There are two types of Version Control Systems, Centralized and Distributed. Centralized Version Control Systems (VCS), such as Subversion, which is a tool that has been trusted to house version revision history, is a centralized point, which can lead to a loss of work if we lose network connectivity.

Distributed Version Control Systems (DVCS) get around this problem. Some examples of distributed systems, apart from Git, are Mercurial, Bazaar, or Darcs. In these types of tools, clients fully replicate the repository.
