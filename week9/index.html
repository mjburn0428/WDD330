Week 09 Learning Notes: Storage

This week I learned about Local Storage and Session Storage. For Session Storage the data will survive and be there after a page refresh, and for Local Storage a full restart of the browser. Both Storage Objects have the same methods and properties such as setItem(key,value) getItem(key) removeItem(key) clear() key(index) length We can use a plain object to obtain or set keys like this, //This sets the key localStorage.test = 3; // This obtains the key alert( localStorage.test ); // 3 // This removes the key delete localStorage.test; We can also loop over keys. Storage objects are not iterable. We can use an array to loop over them such as, for(let i=0; i<"localStorage.length"; i++) { let key = localStorage.key(i); alert(`${key}: ${localStorage.getItem(key)}`); } We can filter fields which hasOwnProperty such as: for(let key in localStorage) { if (!localStorage.hasOwnProperty(key)) { continue; // skip keys like "setItem", "getItem" etc } alert(`${key}: ${localStorage.getItem(key)}`); } As a note both key and value must be strings. If it were another type such as a number or an object it would get converetd to a string automatically. We can use JSON to store objects localStorage.user = JSON.stringify({name: "John"}); // sometime later let user = JSON.parse( localStorage.user ); alert( user.name ); // John Session storage is used less often than local storage. The properties and methods will be the same but very limtied. Session storage exists within the browser tab. Another tab with the same page will be stored different in a different storage. To sum it all up. Both value and ley need to be strings. Depending on the browser the limit is 5mb. The storage of local and session have no expiration, and finally the data is bound to the orgin of the domain,port,or protocol. Local storage survives a browser restart while session storage survives a page refresh but not when the tab is closed.
